# Appendix D: Data Dictionary

> *Every CSV, parquet, and key data structure in the Tallgrass pipeline — what it contains, where it lives, and what the columns mean.*

---

## Scraper Output CSVs

Five CSV files per biennium, stored in `data/kansas/{legislature}_{start}-{end}/`:

### votes.csv

One row per legislator per roll call. The fundamental input to all analyses.

| Column | Type | Description |
|--------|------|-------------|
| `slug` | string | Legislator identifier (e.g., `rep_smith_john_1`). Prefix: `rep_` = House, `sen_` = Senate |
| `full_name` | string | Display name (e.g., "John Smith") |
| `vote` | string | Vote cast: `Yea`, `Nay`, `Present and Passing`, `Absent and Not Voting`, or `Not Voting` |
| `vote_id` | string | Roll call identifier. Format: `je_YYYYMMDDHHMMSS` (e.g., `je_20250320203513`). KanFocus: `kf_{num}_{year}_{chamber}` |
| `party` | string | `Republican`, `Democrat`, or `Independent` (empty string filled to "Independent" at load) |
| `chamber` | string | `Senate` or `House of Representatives` |
| `district` | string | District number |
| `session` | string | Session identifier (e.g., `2025-2026` or `2024s` for special sessions) |

### rollcalls.csv

One row per roll call. Bill-level metadata for each vote.

| Column | Type | Description |
|--------|------|-------------|
| `vote_id` | string | Matches `votes.csv` (join key) |
| `bill_number` | string | e.g., `HB 2032`, `SB 123`, `SCR 1601` |
| `motion` | string | What was voted on: `Final Passage`, `Emergency Final Passage`, `Amendment`, `Veto`, etc. |
| `short_title` | string | Bill's abbreviated title (may be null for older sessions) |
| `yea` | int | Total Yea votes |
| `nay` | int | Total Nay votes |
| `passed` | bool/null | `True` if passed/adopted, `False` if failed/rejected, null if indeterminate |
| `sponsor_slugs` | string | Semicolon-joined slugs of bill sponsors (e.g., `"sen_tyson_caryn_1; sen_alley_larry_1"`). Empty for committee sponsors or pre-89th sessions |
| `chamber` | string | `Senate` or `House of Representatives` |

### legislators.csv

One row per legislator. The roster for the biennium.

| Column | Type | Description |
|--------|------|-------------|
| `name` | string | Short name (last name or abbreviated) |
| `full_name` | string | Full display name |
| `slug` | string | Unique identifier (`rep_` or `sen_` prefix) |
| `chamber` | string | `Senate` or `House of Representatives` |
| `party` | string | `Republican`, `Democrat`, or `Independent` |
| `district` | string | District number |
| `member_url` | string | URL to legislator's page on kslegislature.gov |
| `ocd_id` | string | OpenStates OCD person ID (`ocd-person/{uuid}`). Empty if roster not synced |

### bill_actions.csv

One row per bill lifecycle event. Available for sessions with KLISS API (89th+).

| Column | Type | Description |
|--------|------|-------------|
| `bill_number` | string | e.g., `HB 2032` |
| `action_code` | string | KLISS action code |
| `chamber` | string | Where the action occurred |
| `committee_names` | string | Semicolon-joined committee names (if applicable) |
| `occurred_datetime` | datetime | When the action occurred |
| `session_date` | date | Session calendar date |
| `status` | string | Action description |
| `journal_page_number` | string | Reference to the official journal |

### bill_texts.csv

One row per bill document. Generated by `tallgrass-text`, not the vote scraper.

| Column | Type | Description |
|--------|------|-------------|
| `bill_number` | string | e.g., `HB 2032` |
| `version` | string | Document version (introduced, enrolled, etc.) |
| `url` | string | PDF download URL |
| `text` | string | Extracted plain text content |
| `supplemental_note` | string | Legislative Research Department summary (if available) |

## Key Analysis Output Files

### IRT Ideal Points (Phase 05/06)

Parquet files in `results/{session}/{run_id}/05_irt/data/`:

**`ideal_points_{chamber}.parquet`**

| Column | Type | Description |
|--------|------|-------------|
| `legislator_slug` | string | Legislator identifier |
| `full_name` | string | Display name |
| `party` | string | Party affiliation |
| `district` | string | District number |
| `chamber` | string | `house` or `senate` |
| `xi_mean` | float | Posterior mean ideal point (liberal < 0, conservative > 0) |
| `xi_sd` | float | Posterior standard deviation (uncertainty) |
| `xi_hdi_3%` | float | Lower bound of 94% HDI |
| `xi_hdi_97%` | float | Upper bound of 94% HDI |

**`bill_params_{chamber}.parquet`**

| Column | Type | Description |
|--------|------|-------------|
| `vote_id` | string | Roll call identifier |
| `alpha_mean` | float | Posterior mean difficulty |
| `beta_mean` | float | Posterior mean discrimination |
| `bill_number` | string | Bill identifier |
| `motion` | string | Vote motion type |

### Canonical Ideal Points (Phase 06)

**`canonical_ideal_points_{chamber}.parquet`** — Same schema as ideal points above, with an additional `source` column indicating which model produced them ("1D IRT", "2D Dim 1", "Hierarchical 2D Dim 1").

### Party Unity and Maverick Scores (Phase 13)

**`maverick_scores_{chamber}.parquet`**

| Column | Type | Description |
|--------|------|-------------|
| `legislator_slug` | string | Legislator identifier |
| `full_name` | string | Display name |
| `party` | string | Party affiliation |
| `unity_score` | float | Fraction of party votes where they voted with their party (0-1) |
| `maverick_rate` | float | Fraction of close party votes where they defected (0-1) |
| `weighted_maverick` | float | Maverick rate weighted by vote closeness (0-1) |
| `n_defections` | int | Total number of defection votes |
| `n_party_votes` | int | Total party-line votes they participated in |

### Network Centrality (Phase 11)

**`centrality_{chamber}.parquet`**

| Column | Type | Description |
|--------|------|-------------|
| `legislator_slug` | string | Legislator identifier |
| `betweenness` | float | Betweenness centrality (0-1) |
| `eigenvector` | float | Eigenvector centrality (0-1) |
| `pagerank` | float | PageRank score |
| `harmonic` | float | Harmonic centrality |
| `cross_party_fraction` | float | Fraction of edges connecting to the other party (0-1) |

### Prediction Results (Phase 15)

**`per_legislator_accuracy_{chamber}.parquet`**

| Column | Type | Description |
|--------|------|-------------|
| `legislator_slug` | string | Legislator identifier |
| `accuracy` | float | Fraction of votes correctly predicted (0-1) |
| `n_votes` | int | Total votes in the test set |
| `n_correct` | int | Correctly predicted votes |

**`surprising_votes_{chamber}.parquet`**

| Column | Type | Description |
|--------|------|-------------|
| `legislator_slug` | string | Legislator identifier |
| `full_name` | string | Display name |
| `party` | string | Party affiliation |
| `chamber` | string | Chamber |
| `vote_id` | string | Roll call identifier |
| `bill_number` | string | Bill identifier |
| `motion` | string | Vote motion |
| `actual` | int | Actual vote (1=Yea, 0=Nay) |
| `predicted` | int | Predicted vote |
| `y_prob` | float | Model's predicted probability of Yea |
| `confidence_error` | float | How confident the model was in the wrong answer |

### Synthesis Unified DataFrame (Phase 24)

**`legislator_df_{chamber}.parquet`** — All metrics from all upstream phases joined on one row per legislator.

| Column | Type | Description |
|--------|------|-------------|
| `legislator_slug` | string | Legislator identifier |
| `full_name` | string | Display name |
| `party` | string | Party affiliation |
| `district` | string | District number |
| `chamber` | string | Chamber |
| `xi_mean` | float | IRT ideal point (canonical) |
| `xi_sd` | float | IRT uncertainty |
| `unity_score` | float | Party unity (CQ) |
| `maverick_rate` | float | Maverick rate |
| `weighted_maverick` | float | Weighted maverick rate |
| `betweenness` | float | Network betweenness centrality |
| `eigenvector` | float | Network eigenvector centrality |
| `loyalty_rate` | float | Clustering loyalty |
| `accuracy` | float | Prediction accuracy |
| `PC1`, `PC2` | float | PCA scores |
| `UMAP1`, `UMAP2` | float | UMAP coordinates |
| `posterior_mean` | float | Beta-Binomial posterior loyalty |
| `shrinkage` | float | Bayesian shrinkage factor |
| `xi_mean_percentile` | float | Ideology percentile rank (0-1) |
| `betweenness_percentile` | float | Centrality percentile rank (0-1) |
| `accuracy_percentile` | float | Accuracy percentile rank (0-1) |
| `cluster_k2`, `cluster_k3` | int | Cluster assignments at k=2, k=3 |

*Note: Columns are present only if the corresponding upstream phase produced output. Missing phases result in null columns.*

### Dynamic IRT (Phase 27)

**`dynamic_ideal_points_{chamber}.parquet`** — One row per legislator per biennium.

| Column | Type | Description |
|--------|------|-------------|
| `legislator_slug` | string | Legislator identifier |
| `full_name` | string | Display name |
| `party` | string | Party affiliation |
| `biennium` | string | e.g., `2025-2026` |
| `legislature` | int | e.g., 91 |
| `xi_mean` | float | Dynamic ideal point for this biennium |
| `xi_sd` | float | Posterior uncertainty |
| `total_movement` | float | Sum of absolute shifts across all bienniums served |
| `net_movement` | float | First-to-last shift |

## Key Identifiers

| Identifier | Format | Example | Used For |
|-----------|--------|---------|----------|
| `slug` / `legislator_slug` | `{chamber}_{last}_{first}_{n}` | `rep_smith_john_1` | Primary legislator ID |
| `vote_id` | `je_YYYYMMDDHHMMSS` | `je_20250320203513` | Primary roll call ID |
| `ocd_id` | `ocd-person/{uuid}` | `ocd-person/a1b2c3d4-...` | Cross-biennium legislator matching |
| `run_id` | `{legislature}-{YYMMDD}.{N}` | `91-260318.1` | Pipeline run identifier |
| `bill_number` | `{prefix} {number}` | `HB 2032`, `SB 123` | Bill identifier |
